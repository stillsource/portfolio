# Palettes CIELAB & Générateur de Contenu Astro Implementation Plan

## Overview
This plan outlines the implementation of metadata extraction and Markdown content generation for the `solenya-sync` Go tool. We will extract EXIF data, compute perceptually accurate color palettes using CIELAB space, and generate Astro-compatible Markdown files.

## Scope Definition (CRITICAL)
### In Scope
- EXIF extraction using `github.com/evanoberholster/imagemeta`.
- Single-image and roll-level CIELAB palette aggregation.
- Refactoring existing palette logic from `internal/processor` to `internal/metadata`.
- Markdown generation in `internal/markdown` following the Astro `rollSchema`.
### Out of Scope (DO NOT TOUCH)
- kDrive API implementation (belongs to another ticket).
- Main sync orchestration logic (beyond providing the components).
- Frontend changes.

## Current State Analysis
- **Existing Logic**: `internal/processor/processor.go:95` implements K-means clustering in Lab space.
- **Astro Schema**: `src/types/content.ts` defines the expected frontmatter.
- **Missing Parts**: EXIF extraction, roll-level aggregation, and Markdown writer.
